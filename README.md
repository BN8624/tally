# Tally

회계프로그램의 전체 매입매출장 엑셀 한 개를 읽어 부가가치세 신고 검토용 월별 집계표를 만드는 로컬 데스크톱 앱입니다. 계산 규칙은 `TALLY_CANON.md`를 따릅니다.

## 설치와 실행

Python 3.11 이상이 필요합니다.

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -e .
python app.py
```

원본 파일은 사용자 PC에서만 처리되며 서버 업로드나 외부 API를 사용하지 않습니다.

## 지원 입력 형식

회계프로그램에서 내려받은 `.xlsx` 또는 `.xlsm` 전체 매입매출장을 지원합니다. 필수 열은 구분, 전표일자, 거래처, 품명, 공급가액, 부가세, 합계, 매입/매출 유형, 계정코드, 계정과목, 카드사명, 카드번호입니다.

전표일자가 실제 날짜인 상세 거래만 집계합니다. 월계, 누계, 분기 누계, 반기 누계, 합계, 제목, 공백 행은 제외합니다. 필수 열이나 금액 형식이 잘못되면 문제가 발생한 시트와 열 또는 행을 표시하고 처리를 중단합니다.

중복된 `Code` 열이 있으면 `계정과목` 바로 앞의 `Code`를 계정코드로 사용합니다.

## 업체 설정

첫 화면의 `새 업체 설정 추가`에서 다음 값을 저장합니다.

- 업체명.
- 146번 표시 명칭인 상품 또는 음식재료.
- 고정자산으로 처리할 계정코드.
- 업체별 계정코드 예외.
- 차량, 거래처, 사업 무관 지출 후보 키워드.

업체 설정은 로컬 `data/companies.json`에 저장되며 기본 분류 규칙보다 우선합니다.

## 불공 검토

원본 불공과 과세 중 불공 후보만 검토 화면에 표시됩니다. 후보는 자동으로 불공 확정되지 않습니다.

- 개별 거래에 판정, 불공 사유, 메모를 입력할 수 있습니다.
- 같은 계정코드와 후보 근거를 가진 거래에 판정을 일괄 적용할 수 있습니다.
- 판단 보류, 원본 불공, 신규 불공 후보만 필터링할 수 있습니다.
- 불공으로 확정할 때는 불공 사유 입력이 필요합니다.

## 결과 엑셀

결과 파일에는 다음 시트가 생성됩니다.

- `집계표`는 위쪽 매입, 아래쪽 매출, 불공 차감, 면세, 카드·현영 보조 집계를 월별로 표시하며 금액이 없는 항목은 만들지 않습니다.
- 매입의 카면·카영·현면·현영은 공제 합계에 넣지 않고 유형별 보조표로 표시합니다.
- 세금계산서 매출 계정이 여러 개면 전체 세금계산서 매출표 옆에 계정별 공급가액을 나눠 표시합니다.
- `불공검토`는 후보 근거와 최종 판정, 불공 사유, 메모를 보존합니다.
- `거래상세`는 원본 시트·행, 카드사·카드번호, 계정 분류, 최종 유형 등 재집계에 필요한 값을 보존합니다.
- `미분류`는 정본 규칙으로 분류할 수 없는 과세·불공 매입을 표시합니다.
- `검산`은 건수와 금액, 월별 합계, 미분류와 보류 상태를 표시합니다.

## 검산 실패 시 확인

다음 중 하나라도 있으면 검산 완료로 표시하지 않습니다.

- 상세 거래 누락 또는 중복.
- 공급가액, 세액, 합계금액 또는 월별 합계 차이.
- 계정 미분류 거래.
- 불공 판단 보류 거래.

결과 화면에서 실패 항목을 확인하고 업체 설정이나 불공 판정을 수정한 뒤 다시 저장합니다. 실패 상태의 검토용 엑셀도 사용자가 확인 후 저장할 수 있습니다.

## 알려진 제한사항

- 입력 열을 사용자가 직접 연결하는 자유 형식 매핑은 지원하지 않습니다.
- 홈택스, 회계프로그램, 전자신고와 직접 연동하지 않습니다.
- OCR과 AI 자동 불공 확정 기능을 사용하지 않습니다.
- 원본 거래를 수정하거나 삭제하지 않습니다.
- 거래상세에 직접 분류되지 않은 의제매입세액과 공통매입 배분을 추정하지 않습니다.
- 플랫폼별 매출, 택시 경감·감차·복지처럼 업체별 수기 계산식은 범용 집계표에 자동으로 넣지 않습니다.
