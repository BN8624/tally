# 매입매출장 부가가치세 월별 집계 앱 기획 정본 v1.0

## 1. 제품 정의

회계프로그램에서 내려받은 **전체 매입매출장 엑셀 한 개**를 넣으면, 매입과 매출을 자동으로 나누고 월별로 집계하여 현재 손으로 작성하는 **부가가치세 신고 검토용 집계표**를 만드는 앱이다.

최종 집계표의 배치는 다음과 같다.

* 위쪽: 매입
* 아래쪽: 매출
* 면세: 과세와 섞지 않고 별도 집계
* 불공: 과세 매입에 먼저 포함한 뒤, 별도 표시하여 차감
* 카드매출·현금영수증 매출: 총매출에 포함하되, 확인용으로 다시 별도 표시

앱의 목적은 부가가치세 신고서를 직접 작성하는 것이 아니라, **신고서에 입력할 매입·매출 금액을 월별로 펼쳐 검토할 수 있는 작업표를 자동 생성하는 것**이다.

---

## 2. 입력 파일

### 2.1 입력 형식

입력은 회계프로그램에서 엑셀로 변환한 `전체 매입매출장`이다.

현재 원본에서 사용하는 주요 열은 다음과 같다.

| 열         | 용도                  |
| --------- | ------------------- |
| 구분        | 매입과 매출 분리           |
| 전표일자      | 월별 집계               |
| 거래처       | 거래 확인 및 불공 후보 판단    |
| 품명        | 불공 후보 판단            |
| 공급가액      | 집계                  |
| 부가세       | 집계                  |
| 합계        | 카드매출·현금영수증 매출 및 검산  |
| 매입/매출 유형  | 과세·불공·면세·카과·현과 등 구분 |
| Code      | 계정코드                |
| 계정과목      | 계정 분류               |
| 카드사명·카드번호 | 카드 거래 확인            |

원본은 정해진 회계프로그램 형식을 사용하므로 별도의 열 연결이나 전처리 화면을 만들지 않는다.

앱 내부에서는 `전표일자`가 실제 날짜인 상세 거래만 집계한다. 월계, 누계, 합계 등 회계프로그램이 추가한 집계 행은 다시 더하지 않는다.

---

## 3. 전체 처리 흐름

```text
전체 매입매출장 불러오기
→ 매입과 매출 분리
→ 매입 유형 분리
→ 과세·불공 계정 분류
→ 과세·불공 안에서 불공 검토
→ 매입 월별 집계
→ 매출 월별 집계
→ 면세 별도 집계
→ 불공 차감
→ 원본 대비 검산
→ 집계표 엑셀 생성
```

# 4. 매입 처리

## 4.1 매입 유형 분리

`구분=매입`인 거래를 원본의 매입 유형에 따라 나눈다.

| 원본 유형 | 처리                 |
| ----- | ------------------ |
| 과세    | 과세·불공 묶음에 포함       |
| 불공    | 과세·불공 묶음에 포함       |
| 면세    | 면세 매입으로 별도 집계      |
| 카과    | 카드 과세매입으로 별도 집계    |
| 현과    | 현금영수증 과세매입으로 별도 집계 |

면세·카과·현과 거래에는 불공 후보 검토를 적용하지 않는다.

---

## 4.2 과세·불공 계정 분류

과세와 불공 거래를 합친 뒤 계정코드에 따라 다음 명칭으로 분류한다.

| 계정코드        | 편집본 분류 명칭  |
| ----------- | ---------- |
| 146         | 상품 또는 음식재료 |
| 156         | 원재료(도급)    |
| 업체의 고정자산 계정 | 고정         |
| 500번대       | 제조경비       |
| 600번대       | 도급경비       |
| 800번대       | 기타         |

146번은 업체별로 하나의 명칭을 선택한다.

```text
업체 A: 146 → 상품
업체 B: 146 → 음식재료
```

앱이 임의로 500번대나 600번대를 다시 세분화하지 않는다.

* 500번대는 모두 `제조경비`
* 600번대는 모두 `도급경비`
* 800번대는 모두 `기타`

불공 거래도 해당 계정 분류에 먼저 포함한다.

예를 들어 813 접대비 불공 거래라면:

```text
계정 분류: 기타
매입 유형: 불공
```

으로 처리한다.

즉, `기타` 집계에 포함되면서 불공 집계에도 다시 표시된다.

---

## 4.3 불공 검토

유형 분리와 계정 분류가 끝난 다음, **과세·불공 묶음 안에서만** 불공 여부를 검토한다.

### 기존 불공 확인

원본에서 이미 `불공`으로 처리된 거래는 모두 검토 화면에 표시한다.

판정:

* 불공 유지
* 과세로 변경
* 보류

### 과세 중 불공 후보 찾기

원본에서는 과세지만 불공 가능성이 있는 거래를 후보로 표시한다.

후보 판단에 사용하는 정보:

* 계정코드와 계정과목
* 거래처
* 품명
* 차량번호나 적요
* 기존에 확정한 업체별 규칙

대표적인 후보:

* 접대비·기업업무추진비 관련 거래
* 비영업용 소형승용차의 주유·수리·타이어·유지비
* 사업과 직접 관련 없는 개인적 지출
* 불공 가능성이 있는 증빙 또는 사용 목적

앱은 후보만 제시하고 자동으로 불공을 확정하지 않는다.

판정:

* 과세 유지
* 불공으로 변경
* 보류

불공으로 확정할 때는 사유를 함께 저장한다.

---

## 4.4 과세·불공 월별 집계

과세와 불공을 합친 금액을 **월별·계정 분류별**로 집계한다.

각 분류마다 다음 세 값을 표시한다.

* 매수
* 공급가액
* 세액

예시:

| 월  | 매수 |        공급가액 |         세액 |
| -- | -: | ----------: | ---------: |
| 4월 |  4 |   5,009,064 |    500,906 |
| 5월 | 15 |  98,651,954 |  9,865,196 |
| 6월 |  4 |  69,405,654 |  6,940,566 |
| 계  | 23 | 173,066,672 | 17,306,668 |

같은 형태의 표를 다음 분류별로 만든다.

* 상품 또는 음식재료
* 원재료(도급)
* 고정
* 제조경비
* 도급경비
* 기타

---

## 4.5 매입 요약 계산

### 일반매입

고정을 제외한 과세·불공 계정 분류를 합한다.

```text
일반매입
= 상품 또는 음식재료
+ 원재료(도급)
+ 제조경비
+ 도급경비
+ 기타
```

### 세금계산서 매입계

```text
세금계산서 매입계
= 일반매입 + 고정
```

### 카드매입

```text
카드매입
= 카과 + 현과
```

카과와 현과는 각각 월별로 매수·공급가액·세액을 집계하고, 두 유형을 합한 카드매입 합계도 표시한다.

### 과세 매입 총계

```text
과세 매입 총계
= 세금계산서 매입계 + 카과 + 현과
```

여기에는 불공 거래가 아직 포함되어 있다.

---

## 4.6 불공 별도 집계와 차감

최종 불공으로 판정된 거래를 월별로 다시 집계한다.

| 월  | 불공 매수 | 불공 공급가액 | 불공 세액 |
| -- | ----: | ------: | ----: |
| 4월 |       |         |       |
| 5월 |       |         |       |
| 6월 |       |         |       |
| 계  |       |         |       |

불공은 과세 매입에서 처음부터 제외하지 않는다.

```text
1. 과세·불공 계정 분류에 포함
2. 과세 매입 총계에 포함
3. 불공으로 별도 표시
4. 최종 공제대상 금액에서 차감
```

최종 계산:

```text
과매계(공제대상 과세매입)
= 과세 매입 총계 - 불공
```

집계표에는 공급가액과 세액을 모두 표시한다. 불공 상세표에는 불공 사유도 함께 남긴다.

---

## 4.7 면세 매입

면세 매입은 계정 분류 없이 별도로 월별 집계한다.

| 월  | 매수 | 공급가액 |
| -- | -: | ---: |
| 4월 |    |      |
| 5월 |    |      |
| 6월 |    |      |
| 계  |    |      |

면세에는 세액을 표시하지 않으며 과세 매입 총계에 포함하지 않는다.

---

# 5. 매출 처리

## 5.1 매출 유형 분리

`구분=매출`인 거래를 원본의 매출 유형에 따라 구분한다.

현재 원본에서 확인되는 대표 유형:

* 과세
* 건별
* 카과
* 현과
* 면세가 존재하는 업체는 면세 별도 처리

과세·건별·카과·현과는 모두 과세매출에 포함한다.

---

## 5.2 매출 계정별 월별 집계

매출은 계정과목별로 월별 집계한다.

예:

* 상품매출
* 업체에 존재하는 다른 매출 계정

각 매출 계정마다 다음 값을 표시한다.

* 매수
* 공급가액
* 세액

예시:

| 월  |  매수 |        공급가액 |         세액 |
| -- | --: | ----------: | ---------: |
| 4월 |  30 | 120,822,181 | 12,082,219 |
| 5월 |  35 |  90,062,431 |  9,006,240 |
| 6월 |  36 |  88,844,751 |  8,884,473 |
| 계  | 101 | 299,729,363 | 29,972,932 |

매출 계정 집계에는 과세·건별·카과·현과 거래가 모두 포함된다.

---

## 5.3 카드매출 별도 표시

카드매출은 이미 상품매출 등 매출 총계에 포함되어 있으므로 다시 더하지 않는다.

사진과 같이 확인용 보조표로 월별 카드매출을 따로 표시한다.

기준:

```text
매출 유형 = 카과
```

집계표의 카드매출 보조 영역에는 월별 `합계금액`을 표시한다.

| 월  | 카드매출 합계금액 |
| -- | --------: |
| 4월 |           |
| 5월 |           |
| 6월 |           |
| 계  |           |

카드매출 상세에는 매수·공급가액·세액·합계가 모두 보존된다.

---

## 5.4 현금영수증 매출 별도 표시

현금영수증 과세매출도 총매출에 포함된 부분이므로 다시 더하지 않는다.

기준:

```text
매출 유형 = 현과
```

사진의 `현영` 영역처럼 월별 합계금액을 별도로 표시한다.

| 월  | 현영 합계금액 |
| -- | ------: |
| 4월 |         |
| 5월 |         |
| 6월 |         |
| 계  |         |

---

## 5.5 매출 계산 관계

```text
과세매출 총계
= 과세 + 건별 + 카과 + 현과
```

카드와 현영은 과세매출 총계 안에 들어 있는 부분을 확인하기 위한 별도 명세다.

따라서 다음과 같이 다시 더하면 안 된다.

```text
과세매출 총계 + 카드 + 현영  ← 잘못된 계산
```

필요한 경우 다음 값을 참고용으로 표시한다.

```text
그 밖의 과세매출
= 과세매출 총계 - 카드매출 - 현영매출
```

---

## 5.6 면세 매출

면세 매출이 있는 업체는 과세매출과 분리하여 월별로 집계한다.

| 월  | 매수 | 공급가액 |
| -- | -: | ---: |
| 4월 |    |      |
| 5월 |    |      |
| 6월 |    |      |
| 계  |    |      |

면세 매출에는 세액을 표시하지 않는다.

# 6. 최종 집계표 배치

* 업체명

* 신고 기간
* 예정 또는 확정 구분

예:

```text
2026년 1기 확정

```

연도와 월 범위는 원본 전표일자에서 자동으로 가져온다.

## 6.2 위쪽: 매입 영역

사진과 같은 순서로 배치한다.

### 계정 분류별 월별 표

* 원재료(도급)
* 제조경비
* 도급경비
* 기타
* 고정

각 영역:

```text
월 / (매수) / 공급가액 / 세액
```

```text

고정

```

## 6.3 아래쪽: 매출 영역

* 카드매출 월별 합계금액

사진처럼 카드와 현영은 매출 집계 옆 또는 오른쪽 아래의 보조 영역에 배치한다.

# 8. 매수 계산 기준

기본 매수는 실제 날짜가 있는 상세 거래행의 수다.

* 상세 거래 한 행 = 1매
* 음수 수정 거래도 한 행으로 집계
* 음수 거래의 공급가액·세액·합계는 음수로 합산
* 월계·누계·합계 행은 매수에 포함하지 않음

---

# 9. 검산 기준

앱은 결과를 저장하기 전에 다음 사항을 자동 검산한다.

## 매입 검산

```text
과세·불공 계정 분류 합계
= 원본 과세 + 원본 불공

세금계산서 매입계
= 일반 + 고정

카드매입
= 카과 + 현과

과세 매입 총계
= 세금계산서 매입계 + 카과 + 현과

과매계
= 과세 매입 총계 - 최종 불공
```

## 매출 검산

```text
과세매출 총계
= 과세 + 건별 + 카과 + 현과

카드매출 보조표
= 카과 거래 합계금액

현영 보조표
= 현과 거래 합계금액
```

## 전체 검산

* 원본 상세 거래 건수와 결과 상세 거래 건수 일치
* 공급가액 합계 일치
* 세액 합계 일치
* 합계금액 일치
* 월별 합계 일치
* 거래 누락 없음
* 거래 중복 없음
* 계정 미분류 없음
* 불공 보류 거래 없음

불일치가 있으면 결과를 `검산 완료`로 표시하지 않는다.

# 10. 엑셀 출력

## `집계표`

사진과 같은 최종 부가세 신고 검토용 집계표.

* 위쪽 매입
* 아래쪽 매출
* 면세 별도
* 불공 별도 차감
* 카드·현영 보조 집계

## `불공검토`

* 기존 불공
* 불공 후보
* 최종 판정
* 불공 사유
* 판정 메모

## `거래상세`

원본 거래에 다음 파생 값을 추가한다.

* 매입·매출 구분
* 월
* 계정 분류
* 원본 유형
* 최종 유형
* 불공 여부
* 불공 사유

## `검산`

* 유형별 건수
* 월별 공급가액
* 월별 세액
* 원본 대비 차이
* 누락·중복·미분류 내역

# 11. 현재 샘플 원본 기준 검증값

업로드한 현재 원본은 앱 개발 시 기준 테스트 파일로 사용한다.

| 항목           |    값 |
| ------------ | ---: |
| 실제 날짜가 있는 거래 | 715건 |
| 매출           | 604건 |
| 매입           | 111건 |
| 매출 과세        | 151건 |
| 매출 건별        |   1건 |
| 매출 카과        | 451건 |
| 매출 현과        |   1건 |
| 매입 과세        | 108건 |
| 매입 불공        |   3건 |

현재 업체의 매입 계정 분류:

| 계정                      | 처리 |
| ----------------------- | -- |
| 146 상품                  | 상품 |
| 813·814·815·822·831·833 | 기타 |

현재 원본 불공 3건:

* 접대비 1건
* 차량유지비 2건

이 3건은 `기타`의 과세·불공 총집계에 먼저 포함되고, 불공 집계에 다시 표시된 뒤 과매계에서 차감되어야 한다.

# 12. 완료 기준

앱에 전체 매입매출장을 넣었을 때 다음 결과가 한 번에 나와야 한다.

1. 매입과 매출이 정확히 분리된다.
2. 매입 과세·불공이 지정된 계정 분류로 나뉜다.
3. 불공 후보를 검토하고 최종 확정할 수 있다.
4. 과세 매입은 불공을 포함해 집계된 뒤 불공이 별도로 차감된다.
5. 면세 매입과 면세 매출은 별도로 집계된다.
6. 매입은 집계표 위쪽, 매출은 아래쪽에 배치된다.
7. 매출 카드와 현영은 총매출에 포함된 상태로 별도 보조표에 표시된다.
8. 모든 항목이 월별 매수·공급가액·세액 또는 합계금액으로 표시된다.
9. 원본과 결과의 건수와 금액이 일치한다.
10. 최종 결과가 현재 수기로 작성하는 사진의 집계표를 대체한다.

> **정본 제품 정의:** 전체 매입매출장을 넣으면 매입과 매출을 부가세 신고 검토 구조에 맞게 월별 집계하고, 불공·면세·카드·현영을 중복 없이 별도 표시하여 수기 집계장을 자동 생성하는 앱.
